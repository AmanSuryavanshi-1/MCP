Introduction to Model Context Protocol

The Model Context Protocol (MCP) is an open standard designed to simplify and standardize how AI models, particularly Large Language Models (LLMs), connect to external data sources, tools, and services. Think of it as a universal adapter, like USB-C for AI, enabling seamless communication and interaction.

What is MCP?

MCP is an open protocol meticulously designed to standardize and facilitate seamless integration between Large Language Model (LLM) applications and external data sources, tools, and services. Introduced by Anthropic, MCP acts as a universal language, enabling AI agents to interact securely and efficiently with a wide array of external systems.

This standardization is crucial for building robust, secure, and highly modular AI architectures, allowing AI models to interface with dynamic, real-world data effectively.

Key Components

  • MCP Servers: Lightweight programs that expose specific capabilities
  • MCP Clients: Embedded within LLM applications to connect to servers
  • Tools & Resources: Functions and data exposed by servers
  • Prompts: Pre-written templates for task guidance

The "USB-C for AI" Analogy

Universal Standardization

Just as USB-C provides a standardized, universal way to connect devices to peripherals, MCP offers a standardized connection for Large Language Models to tap into diverse data sources and external tools.

Before MCP, integrating LLMs required custom solutions for each tool or data source. MCP simplifies this by providing a common interface, allowing any MCP-compliant client to connect to any MCP-compliant server.

Interoperability

Any MCP client can connect to any MCP server, regardless of the specific tool or data source

Discoverability

MCP clients can query servers to understand available tools and how to use them

Plug-and-Play

Developers can rely on standardized protocol instead of custom integration code

Core Benefits of MCP

Standardization

MCP provides a unified interface for AI models to interact with external tools, data sources, and services. This eliminates the "N x M" integration problem where every LLM would need a custom connector for every external tool.

By standardizing the communication protocol, MCP allows developers to build MCP servers that expose specific functionalities, and any MCP-compliant client can then connect to and utilize these functionalities.

Enhanced LLM Capabilities

MCP bridges the gap between LLMs' linguistic prowess and the ability to perform tangible tasks in the real world. By connecting LLMs to tools that can query databases, send emails, manipulate files, or interact with APIs, MCP empowers them to become more than just conversational partners.

This allows for the creation of AI agents that can execute complex workflows, perform personalized outreach, and assist in financial analysis by incorporating dynamic data sources.

Improved Security and User Control

MCP is designed with considerations for data privacy and tool safety. MCP servers can enforce authentication and authorization, ensuring that LLMs only access data and tools for which they have permission.

Users often have to approve tool usage, providing a layer of control over what actions an AI agent can perform. This is particularly important when dealing with sensitive information in healthcare or business domains.

Rapid Prototyping and Ecosystem Growth

MCP simplifies integration, accelerating the development cycle for AI applications. Developers can quickly connect LLMs to various data sources and tools to test new use cases and gather feedback.

The availability of SDKs in multiple languages (Python, TypeScript, Java, etc.) and a growing list of pre-built MCP servers for common services further speeds up development and fosters collaboration.

Understanding MCP Architecture and Components

The Model Context Protocol architecture is designed around a client-server model, enabling AI applications (clients) to interact with external tools and data sources (servers) in a standardized way.

MCP Architecture Components

Component Role Key Responsibilities Examples
MCP Host/Client
The application where users interact with an LLM and initiate MCP requests Manages user interaction, discovers MCP servers, formats requests, handles responses, enforces security AI-powered IDEs (Cursor), chat interfaces (Claude Desktop), custom AI apps
MCP Server
Lightweight program that exposes specific capabilities via MCP Listens for MCP requests, translates to actions, formats responses, declares capabilities Database MCP server, File System MCP server, Calendar MCP server
Resources
File-like data exposed by an MCP Server that can be read by clients Provides contextual information to LLMs, uniquely identified by URIs Internal documentation, code files, database query results
Tools
Executable functions exposed by an MCP Server that an LLM can call Performs actions or retrieves information, defined with clear parameters query_database, send_email, get_weather_forecast
Prompts
Pre-written templates exposed by an MCP Server to guide users Helps users accomplish tasks by providing structured guidance Templates for common queries, command structures for tools

MCP Host/Client

The MCP Host is the application or environment where the user interacts with a Large Language Model. Examples include AI-powered IDEs like Cursor, chat interfaces such as Claude Desktop, or custom AI workflow applications.

Within the host application, the MCP Client component facilitates communication between the LLM and MCP Servers. It discovers available MCP servers, translates requests into MCP-compliant calls, and handles responses.

The MCP client acts as the "AI brain's" interface to the external world, enabling seamless leverage of external capabilities.

MCP Server

The MCP Server is a critical component that acts as a bridge between LLMs and external data sources, tools, or services. It is a lightweight program that exposes specific capabilities through the standardized MCP protocol.

Servers can be developed in any programming language that can print to stdout or serve an HTTP endpoint, with Python, JavaScript, and Go being common choices.

Resources and Adapters

Resources are file-like data that can be read by clients, encompassing API responses, file contents, database query results, or dynamically generated content. They provide contextual information to LLMs.

Adapters refer to the logic within an MCP server that translates between the MCP protocol and specific external systems. The MCP server itself acts as an adapter, bridging the LLM's world with external tools and data.

Example: A PostgreSQL MCP server adapts MCP requests into SQL queries, executes them against the database, and formats the results into MCP-compliant responses.

Messaging and Security

JSON-RPC 2.0 Messaging

MCP utilizes JSON-RPC 2.0 for messaging between clients and servers, providing a lightweight, language-agnostic, and standardized way to structure communication.

  • Requests: Bidirectional messages expecting responses
  • Responses: Results or errors corresponding to requests
  • Notifications: One-way messages requiring no response

Security & Trust

  • User Consent: Explicit approval for data access and operations
  • Data Privacy: Protection of user data with access controls
  • Tool Safety: Approval required for arbitrary code execution
  • LLM Controls: Limited server visibility into prompts

Defining MCP Tools (Python & TypeScript)

Python Example

Create a Python file weather_server.py:

from typing import Any import httpx from mcp.server.fastmcp import FastMCP # Initialize MCP server mcp = FastMCP("WeatherAndCalcServer") # Simple calculator tool @mcp.tool() def add_numbers(a: int, b: int) -> int: """Adds two integers and returns the result.""" return a + b # Weather forecast tool @mcp.tool() async def get_weather_forecast(latitude: float, longitude: float) -> str: """Fetches weather forecast for given coordinates.""" # API implementation here return f"Weather at {latitude},{longitude}" if __name__ == "__main__": # Start server with stdio transport mcp.run(transport="stdio")

TypeScript Example

Create calculatorServer.ts:

import { McpServer, Tool } from '@modelcontext/mcp-sdk'; const server = new McpServer('TypeScriptCalculatorServer'); @Tool({ name: 'multiplyNumbers', description: 'Multiplies two numbers.', }) function multiply(x: number, y: number): number { return x * y; } server.registerTool(multiply); server.start({ transport: 'stdio', }).then(() => { console.log('MCP Server running...'); });

Key Points

  • Use @mcp.tool() decorator in Python or @Tool decorator in TypeScript
  • Type hints and docstrings are used for tool descriptions
  • Async functions are supported for I/O operations
  • Choose transport: stdio for local, http for remote

Starting and Testing Your MCP Server

Starting the Server

# For Python server uv run main.py # Or with plain Python python main.py

The server will start and wait for connections. For stdio transport, it reads from standard input and writes to standard output.

  1. 1 In your MCP client (Claude Desktop, Cursor), ask a question that would trigger your tool
  2. 2 The LLM should recognize the intent and available tool

Solving the "N x M" Integration Problem

The "N times M" integration problem occurs when N different client applications need to interact with M different server-side tools or data sources. Without standardization, each client needs custom integration for each server.

MCP solves this by introducing a standardized protocol, reducing complexity from N×M integrations to N+M implementations. Any MCP-compatible client can connect to any MCP-compatible server.

Example

If a developer creates a new LLM application (new "N"), it can immediately leverage all existing MCP servers ("M") without custom integrations. Similarly, a new MCP server becomes available to all existing MCP clients.
